Surface Display Chunk

Surface Display Chunk token: ``simDisplay''
A display window for a projection surface.
Origin Property token: ``origin''
The position on screen, in pixels, of the window's lower-left corner. The upper left corner of the screen is (0,0).
Size Property token: ``size''
The size of window to open, in pixels.
Pipe Property token: ``pipe''
The display pipe to open on. See the Display System chunk for more info.
View Property token: ``view''
The display can be set to draw the viewer's left or right eye view, or it can be instructed to open a stereo view. If the system can't open a stereo display, it falls back to mono.
Border Property token: ``border''
Should the window have a border or not? Making a window borderless on X depends on having a window manager that is mostly compatible with the Motif Window Manager, mwm.
Corners Property token: ``corners''
This property defines the position of the surface in space. The XYZ coordinates of each corner of the surface are given.
User Property token: ``user''
A Juggler User that controls this display.
Active Property token: ``active''
A display that isn't active is part of the system, but no window will be opened for it and no rendering will occur. One common use for this feature is to define all Surface Displays as inactive, and use one active Simulator Display to view how all the Surfaces fit together.

Surface Display chunk definition

chunk surfaceDisplay "Surface Display" "This is a surface that images are projected onto."
  Name String 1 "Name" "Unique name of an instance of this chunk type"
  origin Int 2 "origin" vj_valuelabels { "X" "Y" }
